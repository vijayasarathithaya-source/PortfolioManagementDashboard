# Portfolio Management Dashboard
## Business Requirement Document (BRD)

### 1. Project Overview

#### Project Name
Portfolio Management Dashboard

#### Objective
Develop a web-based application that allows investors to manage and monitor their investment portfolios. The application should provide portfolio visibility, transaction history tracking, investment maintenance, and secure user authentication.

---

## 2. Scope

### In Scope

- User Authentication
- Portfolio Dashboard
- Investment Management
- Transaction History
- Performance Tracking
- JWT Security
- REST API
- Docker Deployment

### Out of Scope

- Real-time stock market integration
- Portfolio recommendations
- AI investment advisory
- Multi-factor authentication
- Payment gateway integration

---

## 3. User Roles

### Investor

Can:

- Login
- View portfolio
- Add investments
- Edit investments
- View transactions
- Logout

---

## 4. Functional Requirements

### FR-001 User Authentication

#### Description

User should authenticate using username/email and password.

#### Inputs

- Email
- Password

#### Outputs

- JWT Token
- User Profile

#### Acceptance Criteria

- Valid credentials allow login
- Invalid credentials show error
- JWT token generated after login
- Protected APIs require token

---

### FR-002 Portfolio Dashboard

#### Description

Display portfolio summary.

#### Dashboard Widgets

1. Total Portfolio Value
2. Total Investment Cost
3. Total Profit/Loss
4. Total Assets Count

#### Asset Types

- Stocks
- Bonds
- Mutual Funds

#### Display Fields

- Asset Name
- Asset Type
- Quantity
- Purchase Price
- Current Value
- Profit/Loss
- Return %

---

### FR-003 Transaction History

#### Description

Display investment transaction history.

#### Transaction Types

- BUY
- SELL

#### Fields

- Transaction Id
- Asset Name
- Transaction Type
- Quantity
- Price
- Date
- Total Amount

#### Filters

- Date Range
- Asset Type
- Transaction Type

---

### FR-004 Add Investment

#### Description

User can add investment.

#### Fields

- Asset Name
- Asset Type
- Quantity
- Purchase Price
- Current Value
- Purchase Date

#### Validation

- Asset Name mandatory
- Asset Type mandatory
- Quantity > 0
- Purchase Price > 0
- Current Value > 0

---

### FR-005 Edit Investment

#### Description

User can update existing investment.

#### Editable Fields

- Quantity
- Current Value
- Purchase Price

#### Restrictions

- Asset Type cannot be modified after creation

---

## 5. Business Rules

### BR-001

Only authenticated users can access dashboard.

### BR-002

Investment quantity must be positive.

### BR-003

Purchase price cannot be negative.

### BR-004

Current value must be greater than zero.

### BR-005

Profit/Loss Calculation

Profit/Loss =
(Current Value - Purchase Price)
× Quantity

### BR-006

Performance Percentage

((Current Value - Purchase Price)
/ Purchase Price) × 100
